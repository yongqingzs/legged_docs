# AGENTS.md ‚Äî faster_slam (ROS 2 package: `faster_lio`)

## Critical: package name mismatch
- The directory is `faster_slam`, but the ROS package is named **`faster_lio`** (set by `project(faster_lio)` in CMakeLists.txt).
- All `colcon`, `ros2 launch`, and `find_package` commands must use the package name **`faster_lio`**, never `faster_slam`.

## Build
```bash
colcon build --packages-select faster_lio --cmake-args -DCMAKE_BUILD_TYPE=Release -Wno-dev --symlink-install
```
The `--symlink-install` flag is useful during development so config/rviz/launch changes are live.

## Compile-time option
```bash
cmake .. -DWITH_IVOX_NODE_TYPE_PHC=ON   # use PHC iVox instead of default linear iVox
```
This toggles `IVOX_NODE_TYPE_PHC` ‚Äî affects which `IVox` template specialization is compiled in `laser_mapping_ros2.h:38-42`.

## Executables
| Binary | Node name | Purpose |
|---|---|---|
| `run_mapping_online_ros2` | `laser_mapping` | Frontend LIO (ESEKF-based odometry + mapping) |
| `faster_pgo` | `laser_PGO` | Backend PGO (loop closure + iSAM2 optimization) |

Both are launched together via `ros2 launch faster_lio slam.launch.py`.

## Key launch arguments (`slam.launch.py`)
- `rviz:=false` (default) ‚Äî no RViz
- `relocal:=true` ‚Äî localization mode (uses prior map)
- `pgo:=true` ‚Äî enable backend PGO
- `prior_dir:=company` ‚Äî subdirectory under `prior/` for localization mode

## Architecture

### Source tree (static library `faster_lio_lib`)
- `src/laser_mapping.cc` ‚Äî main LIO node (ESEKF, iVox map, gravity alignment, reloc state machine)
- `src/imu_processing.cc` ‚Äî IMU prediction/preintegration
- `src/pointcloud_preprocess.cc` ‚Äî LiDAR type dispatch (Livox/Velodyne/Ouster), deskewing, downsampling
- `src/options.cc` ‚Äî global tunables (`faster_lio::options::NUM_MAX_ITERATIONS`, etc.)
- `src/icp_alignment.cc`, `src/fusion_alignment.cc` ‚Äî initial alignment strategies for reloc mode
- `src/laser_PGO.cc` ‚Äî standalone PGO node (NOT part of `faster_lio_lib`, links it + GTSAM)

### Config
- Default config: `config/mid360.yaml`
- Parameters are namespaced: `laser_mapping.ros__parameters.*` for frontend, `laser_PGO.ros__parameters.*` for backend
- The initial alignment method (`ICP` vs `Fusion`) is set at `laser_mapping.ros__parameters.initial_alignment.method`

### Custom message
- `msg/Pose6D.msg` ‚Äî preintegrated lidar states; generated by `rosidl_generate_interfaces` in the root CMakeLists

## Dependencies (system)
- **GTSAM v4.2.0** (exact version ‚Äî gtsam v4.2.0 from the `4.2.0` branch)
- Eigen3, PCL >= 1.10, glog, yaml-cpp, TBB, gflags
- `livox_ros_driver2`, `ndt_omp` (ROS packages, jazzy branches)

## PGO topic naming gotcha
The `laser_PGO` node subscribes to **horizon**-suffixed topics:
- `/odometry_gra_horizon` (not `/odometry_gra`)
- `/cloud_registered_body_horizon` (not `/cloud_registered_body`)

These are published by `laser_mapping` ‚Äî do not change the PGO subscriptions without confirming.

## Localization / reloc mode
- Requires a prior map at `prior/<prior_dir>/PGO.pcd` and keyframes in `prior/<prior_dir>/keyframes/`
- The `prior/` directory is gitignored
- Reloc mode loads the prior map into iVox and runs a state machine: `WAITING_FOR_DATA ‚Üí GRAVITY_ALIGNING ‚Üí INITIAL_REGISTRATION ‚Üí TRACKING`

## Runtime output
- On shutdown, `run_mapping_online_ros2` saves timing logs to `Log/timing_log.csv` and trajectory to `Log/`
- PGO saves results to `data/PGO_output/` on shutdown (PCD map, optimized poses, occupancy grid, etc.)
- Both `Log/` and `data/` are gitignored
